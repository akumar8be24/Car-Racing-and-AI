## 🩺 System Health & Metrics

<details open>
<summary><kbd>GET</kbd> <code>/health</code> — <b>Basic API Health</b></summary>
<br/>

Returns the core API status and verifies if the IBM Watsonx AI provider is correctly configured.

**Response (200 OK):**
```json
{
  "status": "ok",
  "provider": "watsonx",
  "watsonx_configured": true,
  "hf_model_id": "ibm-granite/granite-3.1-8b-instruct",
  "watsonx_url": "https://us-south.ml.cloud.ibm.com"
}
```
</details>

<details>
<summary><kbd>GET</kbd> <code>/api/v1/metrics/health</code> — <b>Detailed Dashboard Metrics</b></summary>
<br/>

Returns comprehensive system health metrics used by the frontend `HealthConsole` component.

**Response (200 OK):**
```json
{
  "api": {
    "name": "API Gateway",
    "status": "healthy",
    "value": "Online",
    "lastUpdated": "2026-05-11T14:30:00.000Z"
  },
  "latency": {
    "name": "Response Latency",
    "status": "healthy",
    "value": 142,
    "unit": "ms",
    "threshold": 500
  }
}
```
</details>

## 🏎️ Strategy & AI Engine

<details>
<summary><kbd>POST</kbd> <code>/api/v1/strategy/recommend</code> — <b>AI Pit Strategy</b></summary>
<br/>

> [!IMPORTANT]
> **Authentication Required:** You must pass a valid Firebase Bearer token in the `Authorization` header.

Analyzes an array of lap telemetry and returns a natural-language strategy narration generated by IBM Granite.

**Request Body:**
```json
{
  "circuit": "Monza",
  "driver": "VER",
  "laps": [
    {
      "lap": 1,
      "lap_time_s": 80.5,
      "tyre_wear_pct": 30,
      "gap_ahead_s": 1.5
    }
  ]
}
```

**Response (200 OK):**
```json
{
  "action": "STAY OUT — PREPARE WINDOW",
  "confidence": 95.0,
  "evidence": [
    "Tyre life and pace stability favor extending stint."
  ],
  "confidence_decomposition": {
    "data_quality": 96.8,
    "model_certainty": 94.2,
    "stability": 95.1,
    "regret_bound": 0.15
  }
}
```
</details>

<details>
<summary><kbd>POST</kbd> <code>/api/v1/chat/explain</code> — <b>Race Engineer Copilot Chat</b></summary>
<br/>

Ask free-form questions about the race state to the IBM Granite LLM.

**Request Body:**
```json
{
  "messages": [
    {"role": "user", "content": "Why did we pit early?"}
  ],
  "telemetry_context": {"circuit": "Monza", "wear": 85}
}
```

**Response (200 OK):**
```json
{
  "reply": "The undercut was highly favorable due to a rapid 3% drop in track temperature, making your soft compounds fall off the cliff."
}
```
</details>

## 📡 Live Telemetry Stream (WebSocket)

<details open>
<summary><kbd>WS</kbd> <code>/api/v1/stream/telemetry</code></summary>
<br/>

This WebSocket endpoint provides zero-latency push events to the frontend dashboard. 

> [!NOTE]
> The frontend automatically implements exponential backoff reconnection and a 30-second ping/pong heartbeat to keep the stream alive.

**Client Sends Heartbeat:**
```json
{
  "type": "ping",
  "timestamp": 1715422200000
}
```

**Server Broadcasts Telemetry:**
```json
{
  "type": "telemetry",
  "timestamp": "2026-05-11T14:30:00.000Z",
  "lap": 27,
  "driver": "VER",
  "speed": 285,
  "tyre_compound": "soft",
  "tyre_wear": 85.2
}
```
</details>

## 🚦 Error Handling & Rate Limits

> [!WARNING]
> The API enforces rate limiting per client IP address. The default is **120 requests per minute**.

All standard endpoints return errors in the following consistent format:

```json
{
  "detail": "Error message describing what went wrong"
}
```

| HTTP Status | Meaning | Action Required |
|-------------|---------|-----------------|
| `400` | Bad Request | Check your JSON payload syntax and fields. |
| `401` | Unauthorized | Ensure your Firebase token is valid and not expired. |
| `429` | Rate Limited | Slow down requests. Check `X-RateLimit-Remaining` headers. |
| `500` | Server Error | Internal backend crash. Check Code Engine logs. |
| `503` | IBM Cloud Error | Watsonx AI might be temporarily unreachable. |
